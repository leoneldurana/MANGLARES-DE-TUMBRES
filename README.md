# MANGLARES-DE-TUMBRES
Blue-Carbon-Manglar-TumbesEste repositorio alberga el código, los datos y los modelos dedicados a la estimación, monitoreo y análisis del Carbono Azul (Blue Carbon) en el ecosistema de manglares de Tumbes, Perú. Los manglares de Tumbes son uno de los sumideros de carbono más eficientes del planeta. El objetivo de este proyecto es utilizar herramientas tecnológicas (como teledetección, machine learning y procesamiento de datos geoespaciales) para cuantificar este almacenamiento y apoyar los esfuerzos de conservación y mitigación del cambio climático.Descripción general del proyectoEste proyecto incluye:Modelos de Machine Learning: Algoritmos entrenados para la estimación de biomasa y carbono orgánico en el suelo (SOC).Procesamiento de Teledetección: Scripts para la integración de imágenes satelitales (Sentinel-2, Landsat y datos SAR de ALOS PALSAR).Análisis Multitemporal: Herramientas para evaluar la dinámica de cobertura del manglar y la pérdida/ganancia de carbono a lo largo del tiempo.Compatibilidad con Python: Entorno preparado para análisis reproducible utilizando librerías geoespaciales avanzadas.Pipelines de Datos Automatizados: Flujos de trabajo listos para procesar datos de inventarios forestales de campo y correlacionarlos con variables climáticas.Componentes PrincipalesModelos de Estimación (estimacion_carbono.py)Un script principal de machine learning que:Procesa variables predictoras (índices de vegetación como NDVI, EVI y datos de radar).Entrena y evalúa modelos de regresión (Random Forest / Gradient Boosting) con datos de campo.Genera mapas de densidad de carbono en formato raster (GeoTIFF).Exporta métricas de validación ($R^2$, RMSE) para asegurar la precisión del modelo.Estructura del proyectoPlaintextblue-carbon-manglar-tumbes/

InstalaciónInstalar dependencias del entorno:Bashpip install -r requirements.txt
Ejecutar el preprocesamiento de datos:Bashpython src/utils_geo.py
DespliegueEjecutar la estimación en el área de estudio (Santuario Nacional Los Manglares de Tumbes)El proyecto está preconfigurado con las coordenadas del Santuario Nacional. Utilice el script principal para generar el mapa de carbono:Bashpython src/estimacion_carbono.py --region tumbes_santuario
⚠️ Nota: El script de ejecución contiene rutas locales predefinidas para el entorno de desarrollo. En producción o entornos compartidos, estas deben gestionarse mediante el archivo config.yaml.Configuración de Fuentes de DatosEl proyecto es compatible y se conecta automáticamente con:Google Earth Engine (GEE): Para la descarga e integración de colecciones satelitales en la nube.Copernicus Open Access Hub: Acceso a imágenes Sentinel de alta resolución.Características principalesIntegración de Datos MultifuenteUsa datos ópticos y de radar (SAR) para penetrar el dosel del manglar y estimar mejor la estructura forestal.Configurado con geometrías específicas para los manglares del norte de Perú.Corrección atmosférica y topográfica automatizada para la región de Tumbes.Funcionalidad de Machine LearningLos scripts heredan configuraciones optimizadas para evitar el sobreajuste (overfitting) en zonas densas.Se utiliza validación cruzada espacial (Spatial Cross-Validation) en lugar de la aleatoria para evitar correlación espacial.Soporta la exportación directa a formatos compatibles con QGIS y ArcGIS.DesarrolloConjunto de tecnologíasPython 3.10+: Lenguaje base para el análisis científico de datos.GeoPandas & Rasterio: Manipulación de datos vectoriales y raster de manera eficiente.Scikit-Learn: Marco para el entrenamiento de los modelos de regresión.Google Earth Engine API: Procesamiento masivo de imágenes satelitales en la nube.Consideraciones de Configuración y Seguridad⚠️ Advertencia: Utilice sus propias credenciales de acceso para Google Earth Engine configurando su token mediante earthengine authenticate. Nunca subas tus claves API o tokens de acceso al repositorio público.Para implementaciones en producción o servidores institucionales:Utilice variables de entorno para almacenar credenciales de servicios en la nube.Asegúrese de validar los datos de campo (inventarios de biomasa) antes de reentrenar los modelos operativos.Establezca copias de seguridad de las capas base raster debido a su gran tamaño de almacenamiento.ContribuyendoEste es un proyecto enfocado en la ciencia de datos climáticos y la conservación. ¡Siéntete libre de hacer un fork, proponer mejoras en los algoritmos o aportar nuevos datos de validación!
